4. Google Ads Asset Manager (ads.jochenreider.com)

4.1 Beschreibung der Applikation

Das Google Ads Asset Manager Tool ist eine private Web-Applikation, die über die offizielle Google Ads API auf ein einzelnes Google Ads-Konto zugreift. Das Tool ermöglicht die Analyse von Kampagnen-Assets und die Generierung von KI-gestützten Optimierungsvorschlägen für Anzeigentexte. Zugriff ist ausschließlich dem Kontoinhaber vorbehalten.

4.2 Authentifizierung – Google OAuth 2.0

Die Applikation verwendet Google OAuth 2.0 zur Authentifizierung. Dabei werden folgende Google-Berechtigungen (Scopes) angefordert:

  • https://www.googleapis.com/auth/adwords – Zugriff auf das Google Ads-Konto zum Lesen und Schreiben von Kampagnendaten. Erforderlich für die Kernfunktionalität des Tools (Asset-Analyse und Optimierung).
  • openid, email, profile – Grundlegende Google-Kontoinformationen zur Authentifizierung und Sitzungsverwaltung.

4.3 Welche Daten werden verarbeitet?

Nach der Authentifizierung werden folgende Daten über die Google Ads API abgerufen und verarbeitet:

  • Google-Konto-E-Mail-Adresse und Profilname (nur zur Authentifizierung)
  • Kampagnendaten: Kampagnennamen, Status, Budget, Gebotsstrategie
  • Anzeigengruppen und Ad Assets (Headlines, Descriptions, Bilder)
  • Performance-Metriken: Impressionen, Klicks, CTR, Kosten, Conversions, ROAS
  • Asset-Performance-Labels: Google's eigene Bewertungen (BEST, GOOD, LOW, LEARNING)
  • Konto-Metadaten: Konto-ID, Name, Währung

4.4 Zweck der Datenverarbeitung

Die abgerufenen Daten werden ausschließlich verwendet für:

  • Anzeige des Kampagnen-Dashboards im Tool
  • Analyse der Asset-Performance auf Kampagnenebene
  • Generierung von KI-basierten Vorschlägen für verbesserte Anzeigentexte
  • Rückschreiben von vom Nutzer explizit genehmigten Änderungen über die Google Ads API
  • Aufrechterhaltung der authentifizierten Benutzersitzung

4.5 Datenspeicherung und -aufbewahrung

Über die Google Ads API abgerufene Daten werden ausschließlich für die Dauer der aktiven Sitzung verarbeitet und nicht dauerhaft gespeichert. OAuth-Token werden serverseitig in einer verschlüsselten Sitzung gespeichert und beim Logout unmöglich gemacht. Es findet keine persistente Speicherung von Google Ads-Daten statt.

4.6 Weitergabe an Dritte

Google Ads-Daten werden nicht verkauft, nicht für Werbezwecke verwendet und nicht an Dritte weitergegeben, außer:

  • Anthropic Claude API: Anzeigentextfragmente (Headlines, Descriptions) werden zur KI-Analyse übermittelt. Es werden keine persönlichen Nutzerdaten oder Kontoinformationen weitergegeben.
  • Google: Kommunikation mit Google OAuth 2.0 und Google Ads API gemäß den Google-Nutzungsbedingungen.
