Google Ads Asset Manager

Jochen Reider — jochenreider.com — kontakt@jochenreider.com

Version 1.0 — March 2026

Tool URL: ads.jochenreider.com

1. Overview

Google Ads Asset Manager is a private, single-user web application that connects directly to the Google Ads API to provide a centralized interface for managing, analyzing, and optimizing Google Ads campaigns. The tool enables the account owner to review asset-level performance data, receive AI-generated optimization suggestions, and apply changes (such as updated ad copy) directly to Google Ads campaigns without manual interaction in the Google Ads UI.

The application is built and operated exclusively by Jochen Reider for managing his own Google Ads account. No third-party clients, agencies, or external users have access to the tool.

2. Business Purpose

The primary purpose of the tool is to streamline the management and optimization of Google Ads campaigns by:

  • Centralizing campaign and asset performance data in a single dashboard
  • Using AI (Large Language Model API) to analyze existing ad assets and generate improved headlines, descriptions, and ad copy
  • Enabling direct write-back of optimized assets to Google Ads via the API, eliminating manual copy-paste workflows
  • Monitoring performance trends and flagging underperforming assets automatically

This tool replaces manual, time-consuming workflows in the Google Ads UI with an automated, data-driven pipeline.

4. Google Ads API Usage

4.1 Authentication

The application uses Google OAuth 2.0 with the following scope:

  • https://www.googleapis.com/auth/adwords — Full access to the Google Ads account for reading and writing campaign data.

Authentication tokens are stored server-side in encrypted sessions. Tokens are never exposed to the frontend client.

4.2 Read Operations

The following data is retrieved from the Google Ads API:

ResourceData RetrievedAccess
CampaignsList of all campaigns, status, budget, bidding strategyRead
Ad GroupsAd groups per campaign, status, targetingRead
Ads & AssetsAll ad assets including headlines, descriptions, images, sitelinksRead
Performance MetricsImpressions, clicks, CTR, conversions, cost, ROAS per asset and campaignRead
Asset LabelsGoogle's asset performance ratings (BEST, GOOD, LOW, LEARNING, PENDING)Read

4.3 Write Operations

The application uses the Google Ads API to perform write operations on the authenticated account. All write operations are explicitly confirmed by the user in the UI before being executed. No automated changes are made without user approval.

OperationDescription
Update Ad AssetsApply AI-generated or manually edited headlines and descriptions to existing Responsive Search Ads (RSAs)
Update Ad StatusPause or enable individual ads or entire ad groups based on performance thresholds
Pin AssetsPin high-performing headlines or descriptions to specific positions within RSAs
Remove Underperforming AssetsRemove individual assets (headlines, descriptions) from RSAs consistently rated LOW
Create New Ad VariationsCreate new RSA variations within existing ad groups using AI-generated copy

4.4 API Access Level Required

Basic Access is required. The tool manages a single Google Ads account (the account owner's own account) and makes programmatic changes to ad assets and campaign settings. Test Account access is insufficient because real campaign data and live ad modifications are required for the tool to function as intended.

5. Data Flow

1. User logs in via Google OAuth 2.0 2. Backend fetches campaign + asset data from Google Ads API 3. Frontend displays performance dashboard 4. User selects campaign / asset for optimization 5. Backend sends asset content to AI API for suggestion generation 6. AI suggestions are displayed to the user 7. User reviews and approves suggestions 8. Backend applies approved changes via Google Ads API (mutate operations) 9. Confirmation is shown to the user

6. Security & Privacy

  • Authentication: Google OAuth 2.0 — only the account owner can log in
  • Transport: All communication over HTTPS (Cloudflare Tunnel + TLS)
  • Token Storage: OAuth tokens stored server-side only, never in the browser
  • Data Retention: No Google Ads data is stored persistently; all data is fetched live from the API per session
  • AI Processing: Ad text fragments are sent to the AI API for suggestion generation only; no personal user data is transmitted
  • Access Control: Single-user application — no multi-tenancy, no shared access
  • No Third Parties: No analytics, tracking, or advertising SDKs are used

7. User Interface

7.1 Login Page

Public landing page describing the tool's purpose, with Google OAuth login button. Includes links to Privacy Policy and Terms of Service.

7.2 Campaign Dashboard

Overview of all active campaigns with key performance metrics (impressions, clicks, CTR, cost, conversions). Sortable and filterable.

7.3 Asset Performance View

Detailed view of all assets for a selected campaign. Shows per-asset performance labels and metrics. Allows selection of assets for AI optimization.

7.4 AI Suggestions View

Displays AI-generated suggestions for improved headlines and descriptions. Side-by-side comparison with current assets. User can approve, edit, or reject individual suggestions before applying them to Google Ads.
